## 2. 표 변환 패턴 (codex가 적용한 형식)

표 안의 정보는 모두 보존하면서 다음 4가지 형식으로 변환했다.

### 2.1 항목/설명 매핑 표 → 굵은 항목 + 불릿

```diff
- | 라벨 | 색상 | 의미 |
- | `bug` | 빨강 #d73a4a | 버그 |
+ - **`bug`**: 버그. 권장 색상은 빨강 `#d73a4a`.
```

### 2.2 시간/단계 표 → 순서 리스트

```diff
- | 0:00~0:05 | 5역할 전체 | 역할 분배 | 역할·명령서 |
+ 1. **0:00~0:05 — 5역할 전체**: 역할 분배. 산출은 역할·명령서.
```

### 2.3 매트릭스 표 (3~5열) → H4 헤더 + 다단계 불릿

```diff
- | 단계 | AI가 도와줄 수 있는 것 | 사람이 반드시 할 것 | 위험 신호 |
- | 1. 이슈 | 본문 요약 | 메인테이너 의도 확인 | RFC 진행 중 |

+ #### 1. 이슈 (Issue)
+ - **AI가 도와줄 수 있는 것**: 이슈 본문 요약, 비슷한 이슈 검색.
+ - **사람이 반드시 할 것**: 진짜로 메인테이너가 원하는 작업인지 확인한다.
+ - **위험 신호**: AI가 "쉬운 이슈"라고 했지만 실제로는 RFC 진행 중일 수 있다.
```

### 2.4 비교 표 (KPT 5분 vs 30분 등) → 절(節) 분리

```diff
- | 항목 | 5분 미니 KPT | 30분 통합 KPT |
- | 목적 | 발표 직후 K/P/T 한 줄 | 6주 패턴 묶기 |

+ #### 5분 미니 KPT (1:25~1:30)
+ 5분 미니 KPT의 목적은 발표 직후 Keep, Problem, Try를 한 줄씩 빠르게 남기는 것이다.
+
+ #### 30분 통합 KPT (1:30~2:00)
+ 30분 통합 KPT의 목적은 6주 전체 패턴, AI 활용 회고, 30일 약속을 하나로 묶는 것이다.
```
