## 5. 핵심 운영 원칙 (요약)

본 콘텐츠 전체에 흐르는 5대 원칙. 강사·멘토·TA 모두 매주 점검.

1. **매주 GitHub에 흔적이 쌓이는 워크숍** — "이해했다"가 아니라 "올렸다/만들었다/추가했다"로 학생 진척 측정.
2. **AI 활용을 매주 분산** — 1회차 5분 → 2회차 15분 → 3회차 12분 → 4회차 40분 → 5회차 30분 → 6회차 12분으로 누적. 4·5회차에 첫 노출이 아니라 본격화.
3. **라이선스를 법률 지식이 아니라 신뢰 구조로** — 3회차에서 LICENSE/CONTRIBUTING/COC/SECURITY/Templates를 "오픈소스 운영의 신뢰 구조" 한 우산 아래 묶어 가르친다.
4. **A/B 트랙 분기 + 평가 통합** — 활동은 갈라지지만 핵심 메시지·LO·평가 기준은 같다. "공통 본문 → A 박스 → B 박스" 3블록 구조.
5. **우선순위는 개념 < 실습 < 회고** — 90분에 다 못 넣으면 개념을 먼저 줄인다. 매 회차 끝 5분 KPT는 비협상.

## 7. 콘텐츠 갱신 / 후속 작업 가이드

### 7.1 회차 강의자료를 수정하고 싶다면

본 콘텐츠는 5인 에이전트 팀(`curriculum-architect`, `pedagogy-designer`, `reference-curator`, `lecture-author × 6`, `content-qa-reviewer`)으로 생성됐다. 하네스가 보존되어 있으므로 다음 트리거로 부분 재실행이 가능하다.

| 수정하고 싶은 것 | 트리거 (Claude Code에서 입력) |
|----|----|
| 마스터 플랜 변경 (학습 목표·서사) | "마스터 플랜에서 X를 Y로 바꿔줘" |
| 운영표 변경 (실습 시간·멘토링 절차) | "2회차 운영표에서 Z 시간을 늘려줘" |
| 자료 추가 / 인용 출처 변경 | "5회차 자료 풀에 W를 추가해줘" |
| 특정 회차 슬라이드 수정 | "{N}회차에서 Q 섹션을 다시 써줘" |
| 전체 검수 다시 | "QA 리포트 다시 작성해줘" |

`oss-edu-orchestrator` 스킬이 자동으로 발동되며, 변경된 산출물은 `_workspace/`(중간) → `outputs/`(최종)으로 흐른다. 기존 통과 회차는 일관성에 영향이 없으면 재검토하지 않는다.

### 7.2 중간 산출물 보존

`_workspace/` 디렉토리는 감사 추적용으로 보존된다. 콘텐츠 생성 단계의 모든 결정·QA 리포트·각 에이전트의 가정과 추정이 거기에 남아 있다.
