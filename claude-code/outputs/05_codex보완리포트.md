## 4. codex의 독립 검토 의견 (P3 후보 — 다음 사이클)

각 회차 codex가 보완 액션 외에 자체 분석으로 제안한 추가 개선점. 본 사이클에는 반영하지 않았으며, 다음 운영 후 학생 피드백과 함께 검토.

### 4.1 1회차
- 슬라이드 변환 가이드(부록 A.2)에 외래어 표기 footnote 반영 위치를 명시 → 슬라이드 제작자 누락 방지
- EX-1B 체크리스트가 아직 "옆 페어 자료 1분" 중심 → 다음 사이클에서 30초 코멘트 청취/공유 항목 반영 검토

### 4.2 2회차
- 슬라이드 변환 시 footnote가 첫 화면 또는 발표자 노트에 유지되는지 확인
- 부록 E.1 자가 검증 문구는 `Pull Request`와 `PR` 표기가 혼재 → 핵심 메시지 표기와 함께 일괄 정비 필요

### 4.3 3회차
- AI 모델 라이선스 사례는 시의성이 강함 → 수업 직전 OSI·프로젝트 공식 출처 재확인을 별도 운영 항목으로
- EX-3B는 A/B 트랙 첫 분기이므로 멘토 배정표에 트랙별 담당을 미리 표시하면 현장 질문 병목 감소

### 4.4 4회차
- EX-4B 시작 직후 2~3분간 팀별 AI 도구 로그인·저장소 접근·fork 가능 여부 멘토 스캔 → 30분 실습 손실 감소
- 1:00~1:13 RAG/MCP/A2A 블록 약어 밀도 높음 → 학생 노트 템플릿에 "한 줄 정의 / 오늘 PR과의 연결" 두 칸 사전 배치

### 4.5 5회차
- 6회차 D-1 사전 미팅에서 팀별 컷 우선순위를 한 표로 모으면 발표 형평성 리스크를 더 빨리 봄
- EX-5B 5역할 로그가 제출물로 남으므로 멘토용 최소 합격 예시 1개를 별도 운영자료로 → 채점 편차 감소

### 4.6 6회차
- 6~8팀 병렬 운영 시 갤러리 워크 10분의 산출물 회수 방식 — A/B 그룹 간 상호 청취 절차 보강
- 발표 자료·프로필 공개가 많아지는 회차 → 개인정보 노출 점검 항목(이메일·전화번호·비밀키·학교 내부 URL)을 학생용 최종 체크리스트에 별도 분리

### 4.7 자료풀
- §1.1 표 `약칭` 컬럼에 코드형 식별자와 정식 서비스명 혼재 → 다음 사이클에서 `ID`와 `표시명` 분리
- MCP/A2A 항목은 §3.4·§6.2·§6.3에 중복 등장 → 명칭·발행처·날짜 기준 단일화
