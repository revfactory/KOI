# 6회차 강의자료: 프로젝트 발표, 오픈소스 커리어, 다음 기여로 이어가기

## 메타데이터

- 일시: 2026-06-04 목 19:00-21:00
- 주제: 최종 발표, GitHub 프로필 정리, AI 활용 회고, 30일 액션 플랜
- 핵심 메시지: 오픈소스 경험은 단순한 수업 결과물이 아니라 커리어에서 공개적으로 검증 가능한 실력의 흔적이다.
- 목표 산출물: 최종 발표 자료, 공개 저장소 또는 외부 기여 링크, 정돈된 GitHub 프로필, AI 활용 회고, 30일 액션 플랜
- 트랙: 발표 형식은 공통으로 운영하되 슬라이드 첫 장에 외부 기여 또는 자체 프로젝트 운영 여부를 명시한다.

## 학습 목표

수강생은 이 회차 후 다음을 수행할 수 있어야 한다.

1. GitHub 프로필, README, pinned repo, contribution graph를 커리어 관점에서 정리한다.
2. 6주 여정을 5분 발표와 5분 질의응답으로 설명한다.
3. AI 활용과 사람이 검증한 부분을 공개 증거로 구분해 제시한다.
4. 강의 종료 후 30일 안에 실행할 다음 기여 1건의 계획을 작성한다.
5. 팀과 개인 회고를 통해 다음 오픈소스 참여 경로를 정리한다.

## 수업 전 준비

팀은 발표 자료 초안을 미리 저장소에 올려 둔다. 발표 자료는 5장 안에서 문제, 6주 기록, AI 활용과 검증, 라이선스와 운영 정책, 배운 점과 다음 기여 약속을 다룬다.

개인은 GitHub 프로필 README 첫 단락을 준비하고, pinned repo 후보를 정한다. 개인 기여 링크와 30일 액션 플랜 후보 저장소도 준비한다.

팀은 AI 활용 회고 초안도 준비한다. 여기에는 AI가 실제로 도운 일, 사람이 검증한 일, 위험했던 일이 들어가야 한다.

## 수업 흐름

1. 지난 5주 동안 만든 공개 흔적을 떠올린다.
2. 서로의 GitHub 프로필을 짧게 보고 첫인상을 공유한다.
3. 개인 GitHub 프로필을 1차 정리한다.
4. 팀별로 5분 발표와 5분 질의응답을 진행한다.
5. 좋은 기여자의 태도와 메인테이너로 성장하는 길을 정리한다.
6. 30일 액션 플랜을 작성한다.
7. 6주 통합 회고를 작성한다.
8. 후속 기여와 포트폴리오 정리 계획을 확정한다.

## 핵심 내용

6회차는 새 개념을 많이 배우는 시간이 아니다. 지난 5주 동안 만든 공개 흔적을 정리하고, 다음 기여를 예약하는 시간이다.

최종 발표는 결과를 자랑하는 시간이 아니라 공개 증거를 해석하는 시간이다. 어떤 문제를 선택했는지, 어떤 저장소나 프로젝트를 다뤘는지, 어떤 Issue와 Pull Request와 Commit을 만들었는지, AI를 어디에 썼는지, 사람이 무엇을 검증했는지 보여준다.

오픈소스 포트폴리오는 말보다 링크가 중요하다. GitHub 프로필, pinned repo, README, Pull Request, Issue, 발표 자료가 서로 연결되어야 한다. 외부 Pull Request가 병합되지 않았더라도 좋은 Issue 작성, 재현, Pull Request 제출, 리뷰 대응 기록은 의미 있는 증거가 된다.

좋은 기여자는 작게 시작하고, 명확히 기록하고, 리뷰에 응답하고, 꾸준히 이어간다. 이 과정의 진짜 종료는 마지막 발표가 아니라 30일 안에 다음 기여 1건을 만드는 것이다.

## 활동

### GitHub 프로필 첫인상 확인

짝을 지어 서로의 GitHub 프로필을 30초 동안 본다. 상대에게 다음 문장을 완성해 전달한다.

```text
이 프로필을 30초 봤을 때, 당신은 ___를 할 줄 아는 사람처럼 보입니다.
```

이 피드백을 바탕으로 프로필 README 첫 단락과 pinned repo 구성을 다듬는다.

### GitHub 프로필 정리

10분 안에 완벽하게 꾸미려 하지 않는다. 다음 항목만 우선 정리한다.

- 내가 무엇을 배우고 만들었는지 보여주는 README 첫 단락
- 수업 산출물 또는 외부 기여와 연결되는 pinned repo
- Issue, Pull Request, 발표 자료, AI 회고 링크
- 직접 하지 않은 일이나 병합되지 않은 Pull Request를 과장하지 않는 표현

### 팀 발표

발표는 5분으로 제한한다. 질의응답은 5분으로 진행한다.

발표는 5장 구조를 권장한다.

1. 우리가 선택한 문제와 트랙
2. 6주 동안 만든 Issue, Pull Request, Commit, Review
3. AI를 어떻게 활용했고 사람이 무엇을 검증했는가
4. 라이선스, 운영 정책, 로드맵
5. 배운 점과 다음 기여 약속

발표에는 다음 8개 항목이 모두 들어가야 한다.

- 우리가 선택한 문제
- 선택한 오픈소스 프로젝트 또는 자체 프로젝트
- 6주 동안 만든 Issue, Pull Request, Commit
- AI를 어떻게 활용했는가
- 사람이 직접 검증한 부분은 무엇인가
- 라이선스와 운영 정책
- 앞으로의 로드맵
- 배운 점

질의응답에서는 동료 질문을 먼저 받고, 부족한 공개 증거가 있으면 링크로 확인한다.

### 30일 액션 플랜

개인은 다음 문장을 완성한다.

```markdown
30일 액션 플랜
==============

나는 2026-07-04까지
[어떤 저장소 또는 프로젝트]에
[어떤 종류의 기여]를
[어떤 AI 도구 또는 협업 방식]을 보조로 사용해
[어떤 검증 절차]를 거쳐
공개 증거 1건으로 남긴다.

증거 URL 후보:
30일 후 확인할 날짜:
```

계획은 구체적이어야 한다. "오픈소스 더 해보기"가 아니라 "문서 오탈자 수정 Pull Request 1건", "버그 재현 Issue 1건", "README 사용법 보강 Pull Request 1건"처럼 확인 가능한 행동으로 쓴다.

### 6주 통합 회고

회고에는 다음 내용을 포함한다.

- 계속 가져갈 습관
- 가장 크게 막혔던 지점
- 다음 30일에 시도할 행동
- AI가 실제로 도운 일
- AI 때문에 위험했던 일
- 사람이 검증해서 바뀐 결정
- 가장 보여주고 싶은 공개 링크

## 멘토링

멘토링에서는 발표 자료, 공개 저장소 또는 외부 기여 링크, AI 활용 회고, 30일 액션 플랜을 확인한다.

확인할 질문은 다음과 같다.

- 발표 자료가 저장소에 공개되어 있는가?
- 공개 저장소 또는 외부 Pull Request 링크가 열리는가?
- AI 활용과 인간 검증이 분리되어 설명되는가?
- 라이선스와 운영 정책이 실제 문서와 연결되는가?
- 30일 액션 플랜이 구체적인 저장소, 행동, 검증 방식, 증거 URL 후보를 포함하는가?
- 개인 프로필에 수업 산출물 링크가 연결되어 있는가?

## AI 활용

AI는 발표 스토리라인, README 문장, 회고 초안, 포트폴리오 문장, 예상 질문을 만드는 데 사용할 수 있다. 사람이 확인해야 할 것은 실제 링크와 발표 내용의 일치, 기여 범위 과장 여부, 라이선스와 운영 설명의 정확성이다.

AI 사용 회고에는 다음을 남긴다.

- 가장 효과적이었던 AI 활용 방식
- AI 출력 중 사람이 폐기한 것
- 사람이 검증해 수정한 결정
- 다음 기여에서 AI를 더 안전하게 쓰기 위한 원칙

## 과제

팀 과제는 최종 발표 자료와 공개 저장소 또는 외부 기여 링크를 제출하는 것이다. AI 활용 회고도 함께 제출한다.

개인 과제는 GitHub 프로필 정리, 개인 회고, 30일 액션 플랜 작성이다.

후속 과제는 30일 안에 다음 기여 1건을 실제 공개 증거로 남기는 것이다.
