핵심 메시지는 오픈소스가 코드 공개를 넘어 AI 시대의 학습, 협업, 커리어, 제품 확산 방식이라는 점이다.

- 학습 목표: 오픈소스 프로젝트의 공개 증거를 읽고, 좋은 후보를 비교하며, 팀 트랙 선택 기준을 설명한다.
- 주요 내용: 과정 목표, 최종 발표 기준, 오픈소스 변화, GitHub 트렌드 읽기, AI와 오픈소스, 최근 인기 프로젝트 분석

- 멘토링: 팀 구성, A/B트랙 후보 선택, 후보별 리스크와 다음 조사 액션 확정

## 프로젝트 트랙 운영

### A트랙: 외부 오픈소스 기여

1. 후보 선정: 관심 프로젝트 3개를 비교하고 기여 가능성을 판단한다. 최소 증거는 후보 분석이다.
2. 기여 범위 확정: README 개선, 번역, 예제 추가, 이슈 재현, 테스트 추가, 작은 버그 수정 중 하나를 선택한다. 최소 증거는 선택 근거와 담당자다.
3. 커뮤니케이션: Issue나 Discussion에서 문제 정의와 기여 의사를 명확히 쓴다. 최소 증거는 외부 Issue/Discussion URL이다.
4. PR 제출: 작은 단위로 변경하고 테스트·문서 증거를 붙인다. 최소 증거는 PR URL 또는 초안이다.
5. 리뷰 대응: 메인테이너 피드백을 정리하고 수정 내역을 남긴다. 최소 증거는 Review 대화와 후속 Commit이다.

외부 PR이 병합되지 않아도 재현, 이슈 작성, PR 제출, 리뷰 대응의 품질을 평가한다.

### B트랙: 자체 AI·오픈소스 운영

1. 문제 정의: 캠퍼스, 연구실, 동아리, 학습, 개발자 도구 등 작은 문제를 정한다. 최소 증거는 문제 정의와 사용자다.
2. 저장소 설계: README, LICENSE, Issue, Project 구조를 만든다. 최소 증거는 공개 저장소 URL이다.
3. 최소 기능 구현: RAG 데모, MCP 서버 예제, 프롬프트 관리 도구, Issue 정리 봇 등 작은 기능을 구현한다. 최소 증거는 Commit과 PR이다.
4. 운영 문서화: CONTRIBUTING, 템플릿, 로드맵, 보안·문의 방식을 정리한다. 최소 증거는 운영 파일이다.
5. 공개 발표: 문제, 기능, 운영 정책, AI 검증, 다음 로드맵을 설명한다. 최소 증거는 발표 자료와 회고다.

기능 규모보다 공개 저장소 운영 방식과 증거의 완성도를 우선한다.

## 평가 루브릭

- 오픈소스 이해도, 20%: 우수한 결과물은 라이선스, 기여 문화, 운영 구조를 프로젝트 사례와 연결해 설명한다. 보완이 필요한 결과물은 용어 설명은 가능하지만 자기 프로젝트 적용 근거가 약하다.
- GitHub 협업, 20%: 우수한 결과물은 Issue, PR, Review, Branch가 작은 단위로 정리되어 있고 대화 기록이 명확하다. 보완이 필요한 결과물은 Commit은 있으나 Issue/PR 흐름이나 리뷰 대응이 부족하다.
- 실제 기여·운영, 25%: 우수한 결과물은 외부 PR 또는 자체 저장소가 실제 사용 가능한 공개 증거를 갖춘다. 보완이 필요한 결과물은 산출물이 로컬 작업이나 발표 설명에만 머문다.
- AI 활용 능력, 20%: 우수한 결과물은 AI 사용 목적, 결과, 인간 검증, 수정 내역이 구체적으로 기록되어 있다. 보완이 필요한 결과물은 AI를 썼다는 언급은 있으나 검증 근거가 없다.
- 발표·문서화, 15%: 우수한 결과물은 README, 발표, 회고, 로드맵이 일관되고 포트폴리오로 연결 가능하다. 보완이 필요한 결과물은 발표는 가능하지만 문서와 공개 증거가 흩어져 있다.

## 운영 준비사항

- GitHub 데모: Fork, Branch, PR, Review, Issue Template을 보여줄 수 있는 샘플 저장소
- 프로젝트 사례: README, Issue, PR, Release, LICENSE가 잘 정리된 오픈소스 사례와 보완이 필요한 사례
- 라이선스 예시: MIT, Apache-2.0, GPL 계열의 차이를 설명할 수 있는 교육용 비교 자료
- AI 도구 경계: AI가 도와줄 수 있는 일과 사람이 검증해야 하는 일을 구분한 체크리스트
- 멘토링 기록: 팀별 트랙, 산출물, 위험, 다음 액션을 누적 관리할 템플릿
- 발표 체크: 최종 발표 항목과 공개 증거 URL을 확인할 루브릭
