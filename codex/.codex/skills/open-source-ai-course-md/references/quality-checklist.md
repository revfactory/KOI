# Quality Checklist

## Source Fidelity

- Course title matches `source.txt`: "AI 시대의 오픈소스 실전: 기여자에서 메인테이너까지".
- Final promise remains visible: after 6 weeks, every team operates one public repository or leaves a real contribution trail in an external open source project.
- Audience remains university and graduate students unless the user changes it.
- Six sessions are preserved with the source topics and schedule.
- Track A and Track B are both represented.
- Weekly mentoring outputs match the source progression.
- Evaluation weights match the source unless explicitly changed.
- Missing reference-link definitions from `source.txt` are flagged or resolved.

## Instructional Quality

- Each document states what learners will be able to do, not only what will be covered.
- Each session includes a concrete practice block and an artifact learners can submit.
- AI usage is paired with human verification.
- License/governance content is connected to repository operation, not treated as abstract legal theory.
- Final deliverables are public, inspectable evidence: issues, PRs, commits, reviews, README, license, contribution guide, AI log, and final presentation.
- Mentoring questions produce decisions or artifacts, not open-ended discussion only.

## Cross-Document Consistency

- The education plan, curriculum, and lecture files use the same dates, session titles, goals, tracks, and deliverables.
- Session assignments accumulate toward the final presentation and public repository/contribution evidence.
- Terms are consistent: Issue, Branch, Commit, Pull Request, Review, Maintainer, Contributor, AI usage log.
- The curriculum table and individual lecture files agree on mentoring outputs.

## Citation and Fact Handling

- Do not add "latest" ecosystem claims without verification from official or primary sources.
- If keeping claims from `source.txt`, preserve citation markers or replace them with verified source links.
- Clearly separate source-grounded content, assumptions, and instructor recommendations.
- Avoid legal advice phrasing; frame license content as educational guidance and require verification for real-world legal decisions.

## Markdown Hygiene

- Use one H1 per file.
- Use tables where they improve scanning.
- Avoid overlong paragraphs.
- Keep heading hierarchy logical.
- Use stable file names and relative links when cross-linking course documents.
- Ensure Korean spacing, punctuation, and terminology are professional and consistent.
